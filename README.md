# 반도체 시료 생산주문관리 시스템 (SampleOrderSystem)

가상의 반도체 회사 "S-Semi"를 위한 콘솔 기반 시료 생산주문관리 시스템입니다. 시료 등록, 주문 접수, 승인/거절, 생산, 출고까지 전체 흐름을 하나의 프로그램에서 관리합니다.

## 시작하기

### 요구 사항

- Python 3.10 이상 (개발/검증 환경: Python 3.14)
- 외부 패키지 불필요 (표준 라이브러리 `sqlite3`만 사용). 테스트 실행 시에만 `pytest`가 필요합니다.

### 실행 방법

```bash
python main.py
```

처음 실행하면 `data/sample_order.db`에 SQLite 데이터베이스가 자동으로 생성됩니다. 프로그램을 종료했다가 다시 실행해도 이전에 등록한 시료·주문 데이터는 그대로 유지됩니다.

## 메인 메뉴 사용법

프로그램을 실행하면 아래와 같은 메인 메뉴가 표시됩니다.

```
==================== 반도체 시료 생산주문관리 시스템 ====================
등록 시료: 2   총 재고: 150   전체 주문: 3   생산라인 대기: 1
[1] 시료 등록   [2] 시료 목록   [3] 시료 검색   [4] 시료 주문   [5] 주문 승인/거절   [6] 생산 라인   [7] 출고 처리   [8] 모니터링   [0] 종료
```

상단 요약 줄은 등록된 시료 수, 전체 재고 합계, 전체 주문 수, 생산 라인에 대기 중인 주문 수를 실시간으로 보여줍니다.

| 번호 | 메뉴 | 설명 |
|---|---|---|
| 1 | 시료 등록 | 새 시료를 시스템에 등록합니다. 시료 ID, 이름, 평균 생산시간, 수율, 초기 재고를 입력합니다. |
| 2 | 시료 목록 | 등록된 모든 시료와 현재 재고를 표 형태로 조회합니다. |
| 3 | 시료 검색 | 이름에 특정 검색어가 포함된 시료만 조회합니다. |
| 4 | 시료 주문 | 고객의 시료 주문을 접수합니다(상태: `RESERVED`). 시료 ID, 고객명, 주문 수량을 입력합니다. |
| 5 | 주문 승인/거절 | 접수 대기 중인 주문을 확인하고 승인 또는 거절합니다. 재고가 충분하면 즉시 `CONFIRMED`로, 부족하면 `PRODUCING`(생산 대기)으로 전환됩니다. 거절 시 `REJECTED`로 전환됩니다. |
| 6 | 생산 라인 | 생산 대기 중인 주문(FIFO)을 확인하고, "다음 생산 처리"를 선택하면 맨 앞 주문의 생산을 완료 처리합니다(재고 반영 후 `CONFIRMED`로 전환). |
| 7 | 출고 처리 | `CONFIRMED` 상태 주문 중 하나를 선택해 출고 처리합니다(`RELEASE`로 전환). |
| 8 | 모니터링 | 상태별 주문 건수(`REJECTED` 제외)와 시료별 재고 상태(여유/부족/고갈)를 확인합니다. |
| 0 | 종료 | 프로그램을 종료합니다. |

### 주문 상태 흐름

```
RESERVED --(거절)--------------------------------------> REJECTED
RESERVED --(승인, 재고 충분)--> CONFIRMED --(출고 처리)--> RELEASE
RESERVED --(승인, 재고 부족)--> PRODUCING --(생산 완료)--> CONFIRMED --(출고 처리)--> RELEASE
```

## 운영 도구

### 더미 데이터 생성 (DummyDataGenerator)

테스트용 시료·주문 데이터를 자동으로 생성해 DB에 채워 넣습니다.

```bash
# 기본값: 시료 10개, 주문 20개 생성
python scripts/dummy_data_generator.py

# 개수 및 대상 DB 지정
python scripts/dummy_data_generator.py --samples 5 --orders 10 --db data/sample_order.db
```

### 데이터 모니터링 도구 (DataMonitor)

SQLite DB의 현재 상태를 콘솔에서 직접 조회하는 관리자 도구입니다. `main.py`의 메뉴 8번(모니터링)과 달리, 테이블/행 데이터를 그대로 확인하거나 실시간 감시(watch)를 할 때 사용합니다.

```bash
python scripts/data_monitor.py
```

메뉴: 테이블 요약(행 수) / 테이블 상세 조회 / 실시간 감시(Ctrl+C로 중단)
